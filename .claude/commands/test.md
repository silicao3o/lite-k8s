# Test Command

테스트를 작성하고 실행합니다.

## Instructions

$ARGUMENTS 에 대한 테스트를 작성하거나 실행합니다.

## Test Writing Guidelines

1. 테스트 이름은 행위를 설명하도록 작성 (예: `shouldReturnErrorWhenInputIsEmpty`)
2. Given-When-Then 또는 Arrange-Act-Assert 패턴 사용
3. 하나의 테스트는 하나의 동작만 검증
4. 테스트는 독립적이고 반복 가능해야 함

## Execution

1. 프로젝트의 테스트 프레임워크 확인
2. 관련 테스트 파일 찾기
3. 테스트 실행
4. 결과 분석 및 보고

실패한 테스트가 있으면 원인을 분석하고 수정 방안을 제시하세요.
